## Codebase Patterns
- ESM project: `"type": "module"` in package.json â€” use `import`/`export`, not `require`/`module.exports`
- TypeScript with NodeNext module resolution â€” file extensions matter for imports
- Use `fileURLToPath(import.meta.url)` + `path.dirname` to get `__dirname` equivalent in ESM
- PRD data lives in `prd.json` at `process.cwd()` with shape `{ userStories: [...] }`
- No external runtime dependencies (no Express, no open package) â€” use Node native modules

- HTML UI is embedded as a template literal in `HTML_PAGE` const in src/index.ts â€” all frontend code lives inline there
- Use `escapeHtml()` in JS to prevent XSS when rendering story content into innerHTML
